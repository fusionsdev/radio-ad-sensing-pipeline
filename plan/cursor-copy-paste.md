# Cursor — ก๊อปวางได้เลย (ไม่ต้องจำ)

---

## เปิด New Chat แล้วว่าง — ปกติครับ

Cursor **ไม่แสดงข้อความต้อนรับ** ให้อัตโนมัติ หน้าใหม่จะว่าง + มีแค่ placeholder ในช่องพิมพ์:

> `Plan, Build, / for commands, @ for context`

**ทำอย่างใดอย่างหนึ่ง:**

| อยากได้ | ทำยังไง |
|---|---|
| เมนูคำสั่ง | **คลิกในช่องพิมพ์** แล้วพิมพ์ **`/`** (slash ตัวเดียว) → รายการ `/status`, `/help` … จะโผล่ |
| พูดตรงๆ | พิมพ์ `วันนี้ pipeline ok ไหม` แล้ว Enter |
| ดู cheat sheet ตลอด | **ปักหมุดไฟล์นี้** ใน editor (คลิกขวา → Pin Tab) |

**ถ้าพิมพ์ `/` แล้วไม่เห็น `/status` `/help`:**

1. ตรวจว่าเปิดโฟลเดอร์ `radio-ad-sensing-pipeline` (แถบบนเห็นชื่อโปรเจกต์)
2. ลอง **Agent** (แผงด้านข้าง) แทน Composer กลางจอ — คำสั่ง custom มักเห็นชัดกว่า
3. Reload: `Ctrl+Shift+P` → พิมพ์ `Reload Window` → Enter
4. หรือพูดภาษาคนได้เลย **ไม่ต้องใช้ `/`**

---

## สำคัญ: คุณไม่ต้องพิมพ์ `/pipeline-ops`

| วิธี | ทำยังไง |
|---|---|
| **พูดตรงๆ (ง่ายสุด)** | พิมพ์ภาษาคน เช่น `วันนี้ระบบ ok ไหม` — agent จะใช้ skill ให้เอง |
| **กด `/` เลือก** | พิมพ์ `/` ใน chat → เลือก `/status`, `/cfpb`, `/help` … |
| **ก๊อปจากไฟล์นี้** | copy บล็อกด้านล่าง → วาง → Enter |

**ไม่ต้อง** พิมพ์ `/pipeline-ops` ทุกครั้ง — มี rule ในโปรเจกต์ช่วยแล้ว

## คำสั่ง `/` ในโปรเจกต์นี้ (พิมพ์ `/` แล้วเลือก)

| พิมพ์ | ทำอะไร |
|---|---|
| `/help` | ดูวิธีใช้ทั้งหมด |
| `/status` | สถานะ pipeline (queue, สถานี, keyword) |

| `/handoff` | สรุปงานก่อนปิด chat |
