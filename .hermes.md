## Architecture (one line)

Ingestor → chunks queue (SQLite) → worker (ASR + LLM) → detections + transcripts → loan classifier (ops) → alerter (Telegram).

## CRITICAL — read DB only inside Docker

```powershell
docker exec radio-worker python -c "import sqlite3; c=sqlite3.connect('/app/data/pipeline.db'); ..."
```

Do **not** trust `data/pipeline.db` on Windows host during ingest.

## Telegram

| Bot | Role |
|---|---|
| **Pipeline alerter** (`radio-alerter`) | Push: first-seen ads, station down, queue drops, periodic report every 3h |
| **Hermes gateway** | Interactive Q&A — use loan-only report for `/pipeline-ops` |

Creds: `.env` → `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID` (never print secrets).

## Response format (Telegram)

1. Action from section 8 (`no_action` / `review_new_loan_candidate` / `rotate_station` / `fix_stream` / `pipeline_problem` / `source_stale_warning`)
2. DB freshness + latest chunk age
3. true_loan counts 1h/6h/24h (skip if stale)
4. Rotate/watch stations if any

Example:

```
Pipeline: OK — loan-only
• Live DB fresh (chunk 4m ago)
• true_loan: 1h=0 / 6h=1 / 24h=3
• Action: no_action
```
