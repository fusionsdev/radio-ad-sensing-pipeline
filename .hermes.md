# Hermes — Radio Ad-Sensing Pipeline

This repo is a **24/7 radio ad-sensing pipeline** on a Windows GPU host (Docker). You are the remote operator via Telegram.

## First action on pipeline questions

Load and follow skill **`/pipeline-ops`** (`.agents/skills/pipeline-ops/SKILL.md`).

For copy-paste Telegram prompts and PowerShell one-liners, read **`plan/telegram-ops-cheatsheet.md`**.

## Non-negotiables

1. **DB:** Query via `docker exec radio-worker` or `.\scripts\pipeline-status.ps1` — **not** host `data/pipeline.db` (stale on Windows Docker).
2. **No Redis** — queue is SQLite `chunks` table.
3. **Read canonical docs:** `AGENTS.md`, `PLAN.md` — do not re-litigate architecture.
4. **Secrets:** Never read/print `.env` or Telegram tokens.
5. **Changes:** Ask before editing `config/stations.yaml`, restarting compose, or mutating production DB.
6. **CFPB collector:** Batch job only — seeds require human verification; see `docs/cfpb-complaint-collector.md`.
